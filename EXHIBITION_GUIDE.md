# Dear Me; Dear You 전시 설명

## 한 줄 소개

**Dear Me; Dear You**는 지금의 마음을 미래의 나 또는 누군가에게 보내는 디지털 편지 서비스입니다.

텍스트, 사진, 그림, 영상으로 편지를 남기고 날짜를 정하면, 약속한 날 이메일로 다시 받아볼 수 있습니다. 편지는 단순히 저장되는 기록이 아니라, 시간이 지난 뒤 다시 확인할 수 있는 기록이 됩니다.

## 기획 의도

우리는 매일 많은 말을 하고 지나가지만, 정작 지금의 감정이나 생각을 오래 남기는 일은 쉽지 않습니다.
이 프로젝트는 현재의 마음을 보관했다가, 정해 둔 날짜에 다시 확인할 수 있도록 만들었습니다.

편지를 쓰는 순간에는 나의 감정을 정리할 수 있고, 시간이 흐른 뒤에는 과거의 나와 다시 마주할 수 있습니다. 누군가에게 보내는 편지는 직접 말하기 어려웠던 응원, 고마움, 위로를 조금 더 부드러운 방식으로 전할 수 있게 합니다.

## 주요 기능

- **미래 편지 보내기**
  원하는 날짜를 정해 미래의 나 또는 다른 사람에게 편지를 보낼 수 있습니다.

- **다양한 편지 형식**
  텍스트 편지뿐 아니라 사진 첨부, 그림 편지, 영상 편지를 지원합니다.

- **이메일 자동 발송**
  설정한 날짜가 되면 작성한 편지가 이메일로 발송됩니다.

- **내 편지함**
  내가 보낸 편지와 받은 편지를 확인하고, 다시 보고 싶은 편지를 저장할 수 있습니다.

- **열린 편지함**
  로그인하지 않아도 익명으로 짧은 편지, 그림, 사진을 남기고 볼 수 있는 공간입니다.

## 체험 방법

1. 회원가입 또는 로그인합니다.
2. 편지 쓰기 화면에서 편지 형식을 선택합니다.
3. 받을 사람과 받을 이메일을 입력합니다.
4. 편지 내용을 작성하고 열람 날짜를 정합니다.
5. 미리보기를 확인한 뒤 편지를 보냅니다.
6. 열린 편지함에서는 익명 편지를 자유롭게 남기거나 읽어볼 수 있습니다.

## 화면별 작동 흐름

### 1. 첫 화면

첫 화면은 사용자가 서비스의 분위기를 가장 먼저 느끼는 진입 화면입니다.
여기서 회원가입, 로그인, 열린 편지함, 내 편지함으로 이동할 수 있습니다.

작동 흐름은 다음과 같습니다.

1. 사용자가 첫 화면에 들어옵니다.
2. 로그인 상태를 확인합니다.
3. 로그인한 사용자는 편지 쓰기나 내 편지함으로 이동할 수 있습니다.
4. 로그인하지 않은 사용자는 회원가입 또는 로그인 화면으로 이동합니다.
5. 열린 편지함은 로그인 없이도 접근할 수 있도록 분리했습니다.

개발적으로는 React Router를 사용해 화면 이동을 관리했습니다.
로그인이 필요한 화면은 보호 라우트로 감싸서, 로그인하지 않은 사용자가 접근하면 자동으로 로그인 화면으로 보내도록 구현했습니다.

### 2. 회원가입 화면

회원가입 화면은 사용자가 서비스를 처음 사용할 때 계정을 만드는 화면입니다.

작동 흐름은 다음과 같습니다.

1. 사용자가 아이디, 이름, 이메일, 비밀번호를 입력합니다.
2. 아이디 중복 확인을 진행합니다.
3. 이메일 형식을 확인합니다.
4. 입력값이 올바르면 서버로 회원가입 요청을 보냅니다.
5. 서버는 비밀번호를 암호화해 데이터베이스에 저장합니다.
6. 회원가입이 완료되면 바로 로그인된 상태로 다음 화면으로 이동합니다.

개발적으로는 서버에서 bcrypt를 사용해 비밀번호를 해시 처리했습니다.
회원가입 후에는 세션을 생성해 사용자가 다시 로그인하지 않아도 바로 서비스를 사용할 수 있도록 만들었습니다.

### 3. 로그인 화면

로그인 화면은 일반 로그인과 편지함 접근용 로그인을 함께 처리합니다.

작동 흐름은 다음과 같습니다.

1. 사용자가 아이디와 비밀번호를 입력합니다.
2. 서버가 계정 정보를 확인합니다.
3. 비밀번호가 맞으면 세션을 생성합니다.
4. 일반 로그인은 메인 흐름으로 이동합니다.
5. 편지함 로그인의 경우 내 편지함 화면으로 이동합니다.

개발적으로는 express-session과 PostgreSQL 세션 저장소를 사용했습니다.
사용자가 새로고침하거나 페이지를 이동해도 로그인 상태가 유지되도록 세션 기반 인증을 적용했습니다.

### 4. Hello 화면

Hello 화면은 로그인 후 사용자가 편지 쓰기나 편지함으로 쉽게 이동할 수 있도록 만든 중간 화면입니다.

작동 흐름은 다음과 같습니다.

1. 로그인한 사용자를 맞이합니다.
2. 편지 쓰기, 내 편지함, 열린 편지함으로 이동할 수 있는 버튼을 제공합니다.
3. 사용자가 목적에 맞는 화면으로 이동합니다.

개발적으로는 사용자 정보 API를 통해 현재 로그인한 사용자의 이름과 이메일 상태를 확인합니다.
이 화면은 기능 선택의 허브 역할을 하며, 전시 상황에서도 사용자가 다음 행동을 쉽게 고를 수 있게 설계했습니다.

### 5. 편지 쓰기 화면

편지 쓰기 화면은 이 프로젝트의 핵심 화면입니다.
사용자는 텍스트, 그림, 영상 형식 중 하나를 선택해 편지를 작성할 수 있습니다.

작동 흐름은 다음과 같습니다.

1. 사용자가 편지 형식을 선택합니다.
2. 받을 사람과 받을 이메일을 입력합니다.
3. 편지 내용을 작성합니다.
4. 사진, 서명, 그림, 영상 등 필요한 미디어를 추가합니다.
5. 열람 날짜를 선택합니다.
6. 오른쪽 또는 왼쪽 미리보기에서 실제 이메일에 가까운 형태를 확인합니다.
7. 보내기 버튼을 누르면 서버에 편지 정보가 저장됩니다.
8. 첨부 파일이 있는 경우 파일은 먼저 저장소에 업로드되고, 편지에는 파일 URL이 연결됩니다.

개발적으로는 편지 형식마다 입력 방식이 다르기 때문에 화면 상태를 세분화해 관리했습니다.
텍스트 편지는 글자 수 제한과 사진 첨부를 지원하고, 그림 편지는 Canvas API를 사용해 그림을 이미지로 저장합니다. 영상 편지는 MediaRecorder API를 사용해 촬영한 영상을 파일로 만들고 업로드합니다.

이미지와 영상은 서버를 거쳐 직접 저장하는 방식이 아니라, 서버에서 업로드용 임시 URL을 발급받은 뒤 Cloudflare R2에 업로드하는 방식으로 구현했습니다.
이렇게 하면 서버에 큰 파일을 오래 들고 있지 않아도 되고, 업로드 흐름도 안정적으로 관리할 수 있습니다.

### 6. 편지 완료 화면

편지 완료 화면은 사용자가 편지를 보낸 뒤 도착하는 화면입니다.

작동 흐름은 다음과 같습니다.

1. 편지 저장이 성공하면 완료 화면으로 이동합니다.
2. 사용자는 내 편지함으로 이동하거나 열린 편지함을 확인할 수 있습니다.
3. 같은 편지를 여러 번 보내지 않도록 버튼 중복 클릭을 막습니다.

개발적으로는 편지 저장 요청이 끝난 뒤에만 완료 화면으로 이동하도록 처리했습니다.
버튼을 빠르게 여러 번 눌렀을 때 같은 요청이 반복되지 않도록 클릭 잠금 처리를 추가했습니다.

### 7. 내 편지함 화면

내 편지함은 사용자가 보낸 편지와 받은 편지를 확인하는 화면입니다.

작동 흐름은 다음과 같습니다.

1. 화면에 들어오면 서버에서 내가 보낸 편지 목록을 가져옵니다.
2. 동시에 내가 받은 편지도 이메일 기준으로 조회합니다.
3. 사용자는 내 편지함과 받은 편지함을 전환할 수 있습니다.
4. 아직 열람 날짜가 지나지 않은 편지는 잠긴 상태로 보입니다.
5. 열람 가능한 편지는 클릭해 편지 보기 화면으로 이동합니다.
6. 다시 보고 싶은 편지는 하트로 저장할 수 있습니다.
7. 열람 전 편지는 필요한 경우 삭제할 수 있습니다.

개발적으로는 편지 목록을 서버에서 받아온 뒤, 열람 가능 여부를 날짜 기준으로 계산해 화면 상태를 나누었습니다.
즐겨찾기 상태는 서버에 PATCH 요청을 보내 저장하고, 화면에서도 바로 반영되도록 처리했습니다.

### 8. 편지 보기 화면

편지 보기 화면은 실제 편지를 열어보는 화면입니다.

작동 흐름은 다음과 같습니다.

1. 사용자가 열람 가능한 편지를 선택합니다.
2. 먼저 편지 봉투 형태의 화면이 나타납니다.
3. 열기 버튼을 누르면 편지 내용이 펼쳐집니다.
4. 텍스트, 그림, 영상 등 편지 형식에 맞게 내용이 표시됩니다.
5. 사용자는 다시 편지함으로 돌아갈 수 있습니다.

개발적으로는 편지 종류에 따라 렌더링 방식을 다르게 처리했습니다.
텍스트는 편지지 형태로 보여주고, 그림은 저장된 이미지 비율을 유지해 표시하며, 영상은 브라우저에서 재생 가능한 형식으로 보여주도록 구성했습니다.

### 9. 열린 편지함 화면

열린 편지함은 로그인 없이도 사용할 수 있는 익명 편지 공간입니다.

작동 흐름은 다음과 같습니다.

1. 사용자가 열린 편지함에 들어옵니다.
2. 전체 공개 편지 목록을 불러옵니다.
3. 사용자는 텍스트, 그림, 사진 중 하나를 선택해 익명 편지를 남길 수 있습니다.
4. 작성된 편지는 공개 목록에 추가됩니다.
5. 사용자는 자신이 작성한 공개 편지를 수정하거나 삭제할 수 있습니다.
6. 관리자는 부적절한 공개 편지를 숨기거나 삭제할 수 있습니다.

개발적으로는 일반 편지와 열린 편지함 편지를 데이터 구조에서 분리했습니다.
개인 편지는 로그인한 사용자와 연결되지만, 열린 편지함은 익명성과 공개성을 중심으로 동작하도록 별도의 API와 테이블 흐름을 사용했습니다.

사진과 그림은 일반 편지와 마찬가지로 업로드 URL을 발급받아 저장소에 업로드하고, 공개 편지 데이터에는 이미지 URL만 저장합니다.

### 10. 관리자 화면

관리자 화면은 전시 운영과 데이터 관리를 위한 화면입니다.

작동 흐름은 다음과 같습니다.

1. 관리자 계정으로 로그인합니다.
2. 회원 목록, 개인 편지 목록, 열린 편지함 목록, 선생님 편지를 확인합니다.
3. 열린 편지함의 공개 여부를 조정할 수 있습니다.
4. 필요한 편지는 선택해서 삭제할 수 있습니다.
5. 선생님 편지는 수정하거나 테스트 발송할 수 있습니다.
6. 예약 날짜가 된 편지를 수동으로 발송할 수도 있습니다.

개발적으로는 관리자 권한을 가진 사용자만 접근할 수 있도록 서버에서 권한 검사를 진행합니다.
관리자 기능은 실수로 일반 사용자가 접근하면 안 되기 때문에, 프론트엔드 라우트 보호뿐 아니라 서버 API에서도 다시 한 번 권한을 확인하도록 구현했습니다.

## 전체 데이터 흐름

### 편지 작성부터 발송까지

1. 사용자가 편지 쓰기 화면에서 내용을 작성합니다.
2. 사진, 그림, 영상이 있으면 먼저 Cloudflare R2에 업로드합니다.
3. 업로드가 끝나면 파일 URL과 편지 내용을 서버로 보냅니다.
4. 서버는 편지 내용, 작성자, 받는 사람, 받을 이메일, 열람 날짜를 데이터베이스에 저장합니다.
5. 매일 정해진 시간에 서버 스케줄러가 발송할 편지를 확인합니다.
6. 열람 날짜가 된 편지는 nodemailer를 통해 이메일로 발송됩니다.
7. 발송 결과는 데이터베이스에 기록됩니다.

### 로그인과 권한 흐름

1. 사용자가 로그인하면 서버가 세션을 생성합니다.
2. 브라우저는 세션 쿠키를 가지고 다음 요청을 보냅니다.
3. 서버는 요청마다 세션을 확인해 사용자를 식별합니다.
4. 편지함, 편지 쓰기, 관리자 화면처럼 보호가 필요한 기능은 로그인 여부를 확인합니다.
5. 관리자 기능은 로그인 여부뿐 아니라 관리자 권한까지 추가로 확인합니다.

### 파일 업로드 흐름

1. 사용자가 사진, 그림, 영상을 추가합니다.
2. 프론트엔드가 서버에 업로드 URL을 요청합니다.
3. 서버는 Cloudflare R2에 업로드할 수 있는 임시 URL을 생성해 돌려줍니다.
4. 프론트엔드는 해당 URL로 파일을 직접 업로드합니다.
5. 업로드가 끝나면 공개 URL을 편지 데이터와 함께 저장합니다.

이 방식은 서버가 큰 파일을 직접 처리하지 않아도 되기 때문에, 영상이나 이미지가 포함된 편지도 더 안정적으로 다룰 수 있습니다.

## 기술적으로 신경 쓴 부분

- **중복 클릭 방지**
  로그인, 로그아웃, 편지 보내기, 열린 편지함 이동처럼 한 번만 실행되어야 하는 버튼은 빠르게 여러 번 눌러도 요청이 반복되지 않도록 처리했습니다.

- **예약 발송 구조**
  사용자가 정한 날짜가 되면 서버 스케줄러가 편지를 찾아 이메일을 발송하도록 만들었습니다.

- **미리보기 화면**
  사용자가 편지를 보내기 전에 실제 이메일과 비슷한 형태로 확인할 수 있도록 미리보기 UI를 만들었습니다.

- **반응형 UI**
  데스크톱과 모바일에서 버튼, 입력창, 미리보기, 편지함이 겹치거나 잘리지 않도록 CSS를 여러 화면 크기에 맞춰 조정했습니다.

- **이메일 검증**
  이메일 오타를 줄이기 위해 주요 도메인과 학교 이메일 형식을 검사했습니다.

- **공개 편지와 개인 편지 분리**
  열린 편지함은 익명 공개 공간이고, 개인 편지는 로그인한 사용자만 볼 수 있는 공간이기 때문에 데이터 흐름과 권한 처리를 분리했습니다.

## 전시 안내 멘트

안녕하세요. 저희 프로젝트는 **Dear Me; Dear You**입니다.
이 서비스는 지금의 마음을 미래의 나 또는 다른 사람에게 보내는 디지털 타임캡슐 편지 서비스입니다.

사용자는 텍스트, 사진, 그림, 영상 중 원하는 방식으로 편지를 작성할 수 있고, 편지가 도착할 날짜를 직접 정할 수 있습니다. 설정한 날짜가 되면 편지가 이메일로 발송되어, 시간이 지난 뒤 과거의 마음을 다시 받아볼 수 있습니다.

또한 열린 편지함에서는 로그인 없이 익명 편지를 남기고 읽을 수 있습니다. 개인적인 편지는 내 편지함에서 관리하고, 모두가 함께 볼 수 있는 짧은 마음은 열린 편지함에 남길 수 있도록 나누어 구성했습니다.

이 프로젝트를 통해 편지가 단순한 메시지가 아니라, 시간이 지난 뒤 다시 확인할 수 있는 기록이 될 수 있다는 점을 보여주고 싶었습니다.

## 강조하고 싶은 점

- 편지를 바로 읽는 것이 아니라, 정해진 시간이 지난 뒤 다시 만난다는 점
- 텍스트 중심의 편지 감성을 해치지 않으면서 사진, 그림, 영상 기능을 더한 점
- 이메일 발송, 예약 날짜, 편지함, 열린 편지함까지 실제 사용 흐름이 이어지도록 구현한 점
- 어두운 화면과 분홍 화면을 구분해 개인 편지와 열린 편지함의 분위기를 다르게 설계한 점

## 짧은 버전

**Dear Me; Dear You**는 미래의 나 또는 누군가에게 편지를 보내는 웹 서비스입니다.
사용자는 텍스트, 사진, 그림, 영상으로 편지를 작성하고 날짜를 정할 수 있으며, 그 날짜가 되면 편지가 이메일로 도착합니다.
또한 열린 편지함을 통해 익명으로 짧은 마음을 남기고 읽을 수 있습니다.
이 프로젝트는 지금의 감정을 시간이 지난 뒤 다시 만날 수 있도록 만드는 디지털 타임캡슐입니다.

## 예상 질문 답변

### Q. 왜 이메일로 보내나요?

편지를 쓴 사실을 잊고 있더라도, 정해진 날 이메일로 다시 받아볼 수 있게 하기 위해 이메일 발송 방식을 사용했습니다.

### Q. 열린 편지함은 왜 만들었나요?

개인적인 편지와 별개로, 전시장에서 누구나 짧은 마음을 남기고 볼 수 있는 공간이 있으면 프로젝트를 더 쉽게 체험할 수 있다고 생각했습니다.

### Q. 편지는 수정할 수 있나요?

타임캡슐이라는 취지를 유지하기 위해 이미 보낸 편지는 마음대로 계속 수정하기보다는, 열람 전 삭제처럼 필요한 최소한의 관리만 가능하도록 설계했습니다.

### Q. 이 프로젝트에서 가장 중요하게 생각한 부분은 무엇인가요?

기능보다도 편지를 쓰는 감정이 먼저 느껴지도록 하는 것이 중요했습니다. 그래서 화면 분위기, 미리보기, 편지함 UI를 편지의 감성과 어울리게 다듬었습니다.
